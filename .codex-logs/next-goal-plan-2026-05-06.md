# Next Goal Plan · 2026-05-06

本计划基于用户最新人工测试反馈。进入下一轮 Goal 前，必须先按这个计划对齐目标和验收口径。

## 目标

把当前版本从“自动化能跑通”修到“核心体验能让用户和朋友继续内测”，优先解决玩家已经明确感知到的低级问题。

## 非目标

- 本轮不宣称真实多人跨机器已验证；用户说等下次强调后再集中测。
- 本轮不扩展新玩法、新怪物种类、新经济系统。
- 本轮不继续用 dev preset 截图替代正常入口体验。
- 本轮不做大规模美术重制，只修比例、清晰度、显示方式和必要的交互闭环。

## 任务包

### P0-A 自动锁定与攻击判定闭环

修复内容：

- 审计锁定目标选择、距离、追击、攻击出手、后撤取消。
- 补玩家可见锁定反馈，避免只有弹窗没有实际帮助。
- 确保普通攻击在合理距离内稳定命中，离谱距离不命中。

验收：

- 自动测试覆盖锁定、追击、取消、攻击命中/未命中。
- 浏览器实机完成一次锁定目标、追击、命中、取消。
- 截图或录制证据显示锁定目标和攻击反馈。

### P0-B 撤离读条稳定性

修复内容：

- 定位撤离点读条闪现来源。
- 修复 HUD 状态抖动或服务端 zone 判定断点。
- 必要时加入边界容差或状态保持，避免站在圈内反复 started/interrupted。

验收：

- 浏览器实机站在撤离点内连续读条，进度条稳定推进。
- 测试边界进出，确认离开才中断，站内不闪。
- 结算仍正常。

### P0-C 画面比例与资产清晰度

修复内容：

- 审计 canvas/CSS/Phaser scaling。
- 修正“简单放大”导致的粗糙和比例不对。
- 调整地图、玩家、怪物、HUD 的显示比例，让资产质量在游戏里体现出来。

验收：

- 桌面窗口截图证明画面比例正确。
- 截图中角色、怪物、地面资产边缘不能明显糊成简单放大。
- UI 不重叠，文字不溢出。

### P0-D 怪物与角色比例体系

修复内容：

- 建立统一比例：玩家、Bot、普通怪、精英怪、Boss。
- 普通怪缩回合理尺寸。
- 精英怪和 Boss 保持层级差异，但不能让所有怪都一样巨大。

验收：

- 同屏或可比较截图显示普通怪、精英怪、Boss 层级明确。
- 不再出现普通怪占屏过大、Boss 与普通怪差异主要靠标签的情况。

### P1-E 黑市复用仓库格子交互

修复内容：

- 黑市“我的装备/可挂售物品”改为格子展示。
- 复用仓库/背包的格子选择、装备位显示、拖拽或点击选中反馈。
- 挂单仍可保留价格确认，但入口不再是列表。

验收：

- 浏览器实机进入黑市，看到格子化物品区。
- 完成一次从格子选择物品、挂单、取消/查看挂单。
- 截图证明黑市和仓库交互风格统一。

### P2-F 多人联机待办

处理：

- 保留为待人工/下一阶段集中验证项。
- 不作为本轮完成口径。

## Subagent 调度方案

### Developer workers

- Worker 1：自动锁定与攻击判定，负责 combat/input/lock assist 文件和对应测试。
- Worker 2：撤离读条稳定性，负责 extract/HUD/zone 判定和对应测试。
- Worker 3：画面比例与怪物比例，负责 Phaser scene、sprite scale、asset display contracts。
- Worker 4：黑市格子化交互，负责 market/stash/inventory UI 复用。

各 worker 写文件范围必须尽量分离，避免互相覆盖。

### Tester workers

- Tester A：按用户反馈做反向验收，不看开发解释，只看玩家实际体验。
- Tester B：跑自动化和浏览器证据收集，确保每个 P0/P1 都有截图或可观察结果。

### Coordinator

- 不直接长期编码。
- 每个 worker 返回后先审计证据，不足则退回。
- 最终按“自动化 + 浏览器 + 玩家视角 + 人工待验”四层 checklist 放行。

## 最终交付清单

结束时必须有：

- 代码提交和 push。
- 每个任务包对应 commit message 写明验收。
- 自动化命令输出摘要。
- 浏览器截图证据路径。
- 明确的人工待验清单。
- 不得把未实测多人联机写成完成。
