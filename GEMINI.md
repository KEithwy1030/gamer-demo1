# Gemini CLI 行为准则

## 核心指令：严禁未经验证的交付

1. **修改即测试**：在修改、新增或删除任何代码后，必须通过自动化手段（Typecheck、Unit Test 或 Playwright 运行时模拟）进行验证。
2. **拒绝口头交付**：严禁在未确认功能正常运行的情况下对用户说“我已经改好了，你去测试吧”。
3. **自主闭环**：如果测试发现 BUG，必须在主 Session 中内部消化并修复，直到验收通过再向制作人汇报最终成果。
4. **视觉验收标准**：对于游戏表现力的修改，必须捕获运行时截图，并通过分析确认视觉效果（如 2.5D 深度、动画状态机）符合预期。
5. **最小化人工介入**：以实现“制作人定方向，Agent 拿结果”为最高目标，消除 Human-in-the-loop 的低效循环。
