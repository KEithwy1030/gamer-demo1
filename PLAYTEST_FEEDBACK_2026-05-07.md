## 严重问题

### 1. 怪物显示比例仍未修复

当前野怪、精英怪、Boss 在游戏画面里的显示比例都过大。

影响：
- 玩家难以判断真实攻击范围。
- 自动锁定和打击判定不好测试。
- 游戏画面显得粗糙、不专业。

下一轮必须优先处理：
- 明确野怪、精英怪、Boss 的目标屏幕尺寸和世界碰撞尺寸。
- 区分视觉尺寸、碰撞半径、攻击接触半径，不再用错误比例互相替代。
- 人工验收必须能直接看出三类怪物比例合理。

### 2. 自动锁定攻击手感不达标

目标体验：
- 玩家点击附近敌人时，即使尚未进入攻击范围，角色也应自动找敌并完成一次攻击。
- 如果玩家距离敌人已经在攻击范围以内，即使玩家移动方向与敌人方向相反，鼠标攻击也应自动朝向敌人打出。
- 锁定的核心不是强制玩家长期面向一个怪，而是让一次攻击意图可靠落到附近/点击的敌人身上。

用户明确偏好的方案：
- 采用“一次攻击意图自动校准目标方向”的方案。
- 不采用长期锁定后移动时始终面向目标的方案。

原因：
- 当前游戏主旋律更偏一对多，不是《黑暗之魂》或《怪物猎人》式一对一锁定战斗。
- 长期面向锁定目标会影响走位、拉扯和一对多混战。
- Web 端和未来移动端都更适合“点击/按下攻击时自动校准目标”的轻量辅助。

### 3. “攻击已出手”提示多余

当前“攻击已出手 / 等待命中判定”类弹窗破坏手感。

下一轮处理：
- 移除“攻击已出手”提示。
- 命中反馈应以明显的扣血数字为主。
- 伤害数字要更大、更清楚、更有爽感。

### 4. 画面资产过于粗糙和像素化

当前游戏画面仍显得粗糙，玩家可明显看到像素方块。

下一轮必须排查：
- 资产源文件是否分辨率过低。
- Phaser 渲染是否启用了不合适的 pixelArt / roundPixels / nearest scaling。
- 单位 sprite 在世界中是否被过度放大。
- 是否需要重新生成或替换更高分辨率资产。

### 5. 河流资产缺失

河流目前没有达到产品级视觉设计要求。

下一轮应补齐：
- 河流视觉资产或程序化水面表现。
- 河流与地图、毒雾/危险区、桥/浅滩的关系。
- 河流不能只是粗糙色块或低质占位。

### 6. HUD / 面板仍未产品化

游戏内 HUD、ID、面板对齐和视觉层级仍然粗糙。

下一轮需要：
- 清理不必要 ID / debug 信息。
- 对齐 HUD 面板布局。
- 提升可读性和视觉层级。
- 以“可上线内测产品”而不是 demo 占位为验收标准。

## 对本轮交付效率的复盘结论

本轮大量时间消耗在 subagent 调度、MCP 启动、浏览器验收不稳定、端口/进程残留和测试脚本修复上。最终真正稳定交付的核心价值集中在撤离读条链路，其他体验项推进不足。

下一轮应避免继续大范围并发 subagent 试错，改为小范围直接修复和人工可见验收：

1. 先修怪物比例和攻击锁定手感。
2. 再修伤害数字反馈。
3. 再修画面资产和河流。
4. 最后统一 HUD 和面板。

每一项都必须有人工可见的验收标准，不能只以脚本通过作为完成标准。
