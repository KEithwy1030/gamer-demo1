# Goal Plan 2026-05-08

本文件是进入 `/goal` 前的执行计划。它基于 `AGENTS.md`、`PITCH.md`、`GDD.md`、`PLAYTEST_FEEDBACK_2026-05-07.md`、`GOAL_HANDOFF_2026-05-07.md`、当前代码检索和地图工具官方资料整理而成。

## 目标

把当前版本从“撤离链路可用、战斗/画面仍像 demo”推进到一个更接近内测产品的可验收版本。

本轮冻结上一轮已改善的锁定攻击手感：默认不改自动锁定/追击/点击攻击逻辑。除非验证发现严重回归，否则任何 worker 不得主动重写 `lockAssist` 或攻击意图逻辑。

## 已确认背景

- 产品锚点：暗黑破坏神 2 方向的手绘暗黑、低饱和、腐烂、铁锈、湿冷雾气；不是像素风、不是卡通、不是高饱和霓虹。
- Demo 1 技术锚点：Phaser 3 + TypeScript + Vite，服务端权威判定；GDD 允许当前阶段用 2D 假装 2.5D，不要求本轮立刻完成真正 2.5D 全资产替换。
- 上一轮人工反馈：撤离读条没有问题；锁定攻击手感已有明显提升，暂时不要再动。
- 当前痛点集中在：伤害数字不明显、怪物比例仍偏大、怪物动作分帧不自然、攻击线条仍像判定 demo、尸毒迷雾粗糙、河流缺少资产化表现、背包拖拽格子偏移和滚动条不美观、地图绘制缺少工业化工具。
- 地图工具判断：Tiled 是本项目优先选项。理由是 Tiled JSON 官方格式支持 orientation/layers/properties，Phaser 官方有 Tiled JSON parser；LDtk 的 JSON 也成熟，但对 Phaser 当前项目不是最低接入成本路径。

参考资料：

- Tiled JSON Map Format: https://doc.mapeditor.org/en/stable/reference/json-map-format/
- Phaser Tiled parser: https://docs.phaser.io/api-documentation/namespace/tilemaps-parsers-tiled
- LDtk JSON overview: https://ldtk.io/docs/game-dev/json-overview/

## 本轮范围

### P0 必交付

1. 伤害数字统一增强
   - 玩家打怪、怪打玩家、流血、尸毒迷雾、河流伤害都必须通过统一反馈链路显示扣血数字。
   - 数字要明显放大、描边增强、漂浮距离更清楚；暴击/流血/环境伤害有区分但不花哨。
   - 去掉“攻击已出手”类提示，以伤害数字和命中反馈替代。

2. 怪物显示比例压缩到当前 50%
   - 以当前代码尺寸为基准：normal 68 -> 34，elite 76 -> 38，boss 90 -> 45。
   - 同步调整影子、血条、光环、标记、名字/皇冠等视觉附属元素，避免附属元素继续显大。
   - 碰撞半径、攻击距离、仇恨距离不默认跟随视觉尺寸砍半；如必须调整，只做小范围验证并写明原因。

3. 取消 demo 攻击线条，替换为命中反馈
   - 移除 `createWeaponVfx` 中偏调试感的攻击线/判定线。
   - 命中后使用更具游戏感的血迹、冲击火花、短暂受击闪白/红、微小位移或缩放反馈。
   - 本轮不做打断系统，不改变服务器战斗结算。

4. 背包拖拽和布局修复
   - 修复战斗背包格子高亮相对鼠标向右偏移的问题。
   - 消除不美观的背包内部滚动条，稳定 10x6 网格显示。
   - 尽量复用大厅“整理行囊”已有的 shared drag helper 和网格计算方式，避免继续维护两套偏移逻辑。

5. 尸毒迷雾视觉和环境扣血反馈
   - 尸毒迷雾不能只是平铺蒙版；增加多层渐变、噪声/流动感、边缘深浅变化。
   - 尸毒扣血、河流扣血都必须从受伤角色身上冒数字。
   - 视觉增强不改变 GDD 的尸毒阶段和服务端权威扣血规则。

### P1 尽量交付

6. 怪物动作分帧小修
   - 在现有 sprite sheet 内优化 idle/move/attack 的帧序和帧率，让站立时不再像机械切帧。
   - 只做现有资源可支持的动作优化；若素材本身不支持 2.5D 斜视角，不强行假装完成。

7. 河流表现小修
   - 在当前程序化地图里把河流从粗线/圆框占位推进到更像水体的表现：暗绿/腐黄水面、边缘泡沫/血污/雾气层。
   - 本轮目标是“比占位明显好”，不是完成最终地图资产系统。

8. Tiled 地图管线最小落地
   - 新增一份地图管线说明或最小样例，明确 Tiled 图层命名、对象层、河流/安全过河点/撤离点/出生点如何映射到现有 `match-layout`。
   - 不在本轮迁移全地图生成系统，避免把可交付修复拖进重构。

## 非目标

- 不重写 Phaser 场景架构。
- 不重写服务端战斗系统。
- 不做完整 2.5D 玩家/怪物资产重制。
- 不迁移完整地图到 Tiled。
- 不继续调整已被用户认可的锁定攻击手感。
- 不用 Chrome DevTools MCP 做正式验收。

6. 最终验收

## Subagent 分工建议

主 session 负责统筹、审查、收口和最终判断；执行层优先使用 GPT-5.4 worker，最终验收使用 GPT-5.5。

- Worker A：伤害反馈链路。只改服务端伤害事件和客户端 damage number，不碰锁定攻击。
- Worker B：怪物比例和动画合同。只改怪物视觉 profile、marker 和相关验证脚本。
- Worker C：背包拖拽和布局。只改战斗背包 UI、CSS、drag validation；参考大厅整理行囊逻辑。
- Worker D：攻击命中反馈、尸毒/河流视觉。注意与 Worker A 的 `feedbackFx.ts` 有文件重叠，必须在 A 合并后再做，或由主 session 串行调度。
- Worker E：Tiled 管线最小方案。偏文档/样例，不阻塞 P0 交付。

每个 worker 提示必须包含：

- 不使用 Chrome DevTools MCP。
- 如需浏览器测试，优先命令行 Playwright 脚本，固定端口，任务结束关闭 browser/server/launcher。
- 不杀全局 Node/Chrome，只清理自己启动的 PID。
- 不 `git add -A` 历史 `.codex-artifacts`。
- 不修改 `AGENTS.md`。
- 不改锁定攻击手感，除非主 session 明确授权。

- 背包 UI 验证脚本：覆盖网格 rect、padding、cursor-to-cell 映射。
- 视觉合同验证脚本：覆盖怪物尺寸、伤害数字字号/描边、攻击线条移除。

## 人工验证清单

这些不能伪装成自动化完成，最终报告必须写“待人工验证”：

- 伤害数字是否足够明显、是否有爽感。
- 怪物缩小后是否仍易读、是否影响攻击范围感知。
- 怪物 idle/move/attack 是否比当前自然。
- 尸毒迷雾和河流是否达到用户预期的美术质感。
- 背包拖拽是否在真实鼠标操作下手感稳定。

## 风险和停下条件

- 若 P0 改动触碰锁定攻击手感，停下给主 session 判断。
- 若要完整替换 2.5D 资产，停下给用户确认资产方向和工期。
- 若地图管线需要迁移现有 `match-layout`，本轮不做，转为下一轮规划。
- 若浏览器/MCP 出现 `Starting MCP servers` 长时间卡住，先做端口和进程归属审计，不继续派发新的 MCP/browser 任务。
- 单步卡住超过 30 分钟，或同一函数连续三次修改仍未通过验证，按 `AGENTS.md` 卡顿规则报告。

- 改了什么。
- 通过了哪些自动验收。
- 哪些是待人工验证。
- 使用了哪些浏览器/MCP/脚本化工具。
- 启动过哪些端口、PID、证据目录。
- browser/server/launcher 是否已关闭，是否还有残留。
