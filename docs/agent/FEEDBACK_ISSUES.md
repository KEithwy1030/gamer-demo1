# 测试反馈问题列表

## Issue 1: 移动端行动摇杆不连贯
- 描述：移动端虚拟摇杆的连续输入体验异常，拖动过程中移动方向和速度不稳定。
- 复现步骤：
  1. 在手机端进入游戏对局。
  2. 持续拖动左侧虚拟摇杆进行移动。
  3. 观察角色移动的连续性与方向响应。
- 期望行为：摇杆拖动期间，角色移动方向与速度连续、稳定，无明显断触、跳变或频繁归零。
- 实际行为：行动摇杆不连贯，移动过程中存在不连续感。

## Issue 2: 移动端交互按键无反应
- 描述：移动端交互按钮无法稳定触发对应行为，影响核心交互操作。
- 复现步骤：
  1. 在手机端进入游戏对局。
  2. 靠近可交互对象或可执行交互场景。
  3. 点击交互按钮。
- 期望行为：点击交互按钮后，应稳定触发对应交互行为，例如拾取、交互或撤离。
- 实际行为：交互按键无反应。

## Issue 3: 手机端移动速度过快
- 描述：移动端虚拟摇杆驱动下的角色移动速度偏快，影响手感和平衡。
- 复现步骤：
  1. 在手机端进入游戏对局。
  2. 使用虚拟摇杆持续移动角色。
  3. 对比桌面端或正常预期下的角色移动节奏。
- 期望行为：移动端角色移动速度应与桌面端手感和游戏节奏基本一致，不应获得异常速度优势。
- 实际行为：手机端移动速度太快。

## Issue 4: 刀剑枪攻击反馈区分度不足
- 描述：当前三种武器的攻击表现差异不明显，玩家无法清晰感知刀、剑、枪的攻击方式区别。
- 复现步骤：
  1. 在游戏内分别使用刀、剑、枪进行攻击。
  2. 观察攻击动作、攻击轨迹、命中视觉反馈。
  3. 对比三种武器的攻击表现差异。
- 期望行为：
  - 刀应体现劈砍感。
  - 剑应体现戳刺感。
  - 枪应体现挥舞感。
  三种武器在攻击动作表现、命中视觉反馈或攻击轨迹上应有明显区分。
- 实际行为：攻击反馈区分不出武器之间的差别，当前手感与视觉表现过于同质化。
