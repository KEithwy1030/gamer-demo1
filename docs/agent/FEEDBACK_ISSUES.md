# FEEDBACK_ISSUES

本文件用于集中记录测试反馈、复现现象、影响范围与后续处理状态。

## 使用规则
- 只记录已经被实际反馈到的问题，不臆测不存在的问题。
- 优先写清“现象”，再补充“场景 / 设备 / 影响”。
- 未确认根因前，不把猜测写成结论。
- 修改和优化该项目的 Agent 应优先参考本文件，按优先级处理，并在处理后回填状态。

## 状态定义
- `new`：新反馈，尚未处理
- `triaging`：正在排查
- `in_progress`：已开始修复
- `fixed_pending_verify`：已修复，待验证
- `closed`：已验证关闭

## 处理约定
- 新增反馈时，直接追加一行，不覆盖原记录。
- 如果多个反馈实际属于同一根因，可在修复阶段合并说明，但保留原始反馈条目。
- 修复后必须补充：处理人、处理提交、验证结果。
