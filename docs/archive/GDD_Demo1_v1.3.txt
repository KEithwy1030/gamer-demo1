1. 项目目标
做一个可运行的 Web 联机 Demo 1，让执行 agent 能直接按文档实现原型。
Demo 1 验证以下核心体验：
三种武器（剑、刀、枪）是否有明显战斗差异
搜 → 打 → 撤 的单局闭环是否成立
掉落、背包、撤离是否能制造紧张感
联机状态下战斗、掉落、死亡、撤离、结算是否能跑通

13.1 珍品定位
珍品是纯撤离计分道具。没有任何战斗属性，不影响战斗；唯一价值是「带出去」——撤离成功后按品质和体积换算得分和金币。死亡掉落后被其他玩家捡走即失去。
13.2 珍品规格
核心设计意图：玩家需要在「带出大体积高价值珍品」和「保留背包空间放更多武器/装备」之间做取舍。珍品本身不增强战斗力，但带出去就是得分和收益。

开发阶段不需要部署到云，全程在本地运行：
终端 1：cd server && npm run dev（启动 Node.js 后端，监听 3000 端口）
终端 2：cd client && npm run dev（启动 Vite 前端，监听 5173 端口）
浏览器打开多个标签页访问 localhost:5173，模拟多人联机
代码修改后自动热更新，不需要重启

19.3 为什么不加 Bot
Demo 阶段加 Bot 需要完整实现 Bot 的移动、战斗、背包、撤离决策逻辑，开发成本等同于再做一套 AI 玩家，会显著拖慢 Demo 进度。测试阶段房主可以自己开多个浏览器标签页模拟多人，足够验证联机逻辑。Bot 列入后续版本迭代计划。
