> 架构宪法管"代码怎么写才是对的"，本文档管"游戏做成什么样才算好"。
> 2026-06-13 由项目所有者口头批准全部内容后落地。

## 0. 项目真实状态判定（先纠偏，再干活）

**作为软件工程，本项目完成约七八成；作为一款游戏，完成不到两成。**

接手本项目的 agent 不许根据"系统齐全、契约脚本全绿、架构整洁"推断项目接近完成。
玩家不玩系统，玩家玩的是看到的、听到的、按下去的那一下。本项目所有玩家可感知的
部分（画面、打击感、反馈仪式、UI 观感）截至 2026-06-13 全部不及格，是当前唯一的
开发主线。

历史教训（为什么会走到这一步）：GPT-5.5 / Claude 4.6/4.7/4.8 历代 agent 的开发循环
都在优化"可自动验证的东西"（契约、事件、架构），因为那里有绿勾可领。手感、观感、
爽点没有脚本打分，于是从来没人做。**不要重复这个失败模式。**

## 1. 验收红线：玩家前三分钟标准

下面四条是本项目的及格线，全部用玩家视角表述。任何体验类交付以此为验收依据。

### 1.1 第一眼（画面）

- 画面读作"一个发生在夜晚的废墟世界"：人物站在地上**有接地影子**，光源（营火/
  火把/撤离点）真实照亮周围，远暗近亮，所有资产像在同一个世界里。
- **玩家屏幕上不许出现任何开发元素**：怪物/宝箱脚下的矢量黄圈、调试按钮、
  方块占位 UI，一律不许见玩家。游戏内提示必须游戏化表达（贴花/着色器光效/
  资产自身的呼吸高亮），不能是几何圈。
- 检验法（截图传播测试）：随手截一张图给陌生人看，反应应该是"这什么游戏？"
  而不是"这是你自己做的吧？"

### 1.2 第一刀（打击手感）

- 命中瞬间必须有完整的反馈链：约 40ms 顿帧 + 受击方 2-4 帧闪白 + 按攻击方向
  击退/硬直 + 命中粒子 + 镜头微震 + 入肉感音效 + 伤害数字从命中点带重量弹出。
- 被打时玩家必须知道疼：屏幕边缘红闪 + 镜头震动。
- 怪物攻击前必须有**动作级前摇**（抬手/红光蓄力），不允许用脚下圈代替。

### 1.3 第一个宝箱（成就感）

- 开出高稀有度物品时游戏要"停下来庆祝"：光效 + 专属音效 sting + 物品名大字弹出。
- 撤离成功必须有结算仪式：本局收益清点、数字滚动、"赚到了"的情绪呈现。

### 1.4 第十分钟（游戏性）

- 玩家应持续在做小决策（再搜一箱还是撤 / 躲这下还是换血），而不是站桩对戳比数值。

## 2. 四条铁律

1. **玩家感觉不到的工作，不算工作。** 排期里每一项必须能回答"玩家在哪一秒感知到它"。
2. **交付凭动起来的证据**（录屏/连续帧序列），不凭"N 个脚本 PASS"。契约绿只是底线，
   不是成果。向用户汇报时禁止把测试通过数当主要成果。
3. **玩家屏幕零开发元素**（见 1.1）。
4. **画面与手感达标（第 1 节四条）之前，冻结一切新系统/新功能/新内容。**
   预留板块不许实装，装备目录不许扩充，新机制不许动工。

---

## 3. "纸片人"问题与整合层职责

单张 AI 生成图片再精致，直接进游戏也是纸片拼贴。原因与对策是固定的：

| 病 | 因 | 治（代码侧职责） |
|---|---|---|
| 悬浮纸片感 | 图片无接地关系 | 引擎绘制接地影（方向与全局光一致） |
| 拼贴感 | 每张图各带烤死的光影/色调 | 全局统一打光、色调分级（color grading）、暗角 |
| 角度打架 | 各资产生成相机角度不同 | 美术圣经锁定同一 2.5D 俯角，资产按它重生成 |
| 细节糊成噪点 | 1024px 源图压到 60px 显示 | 按屏幕实际显示尺寸设计细节密度 |
| 静态精致动态僵硬 | 动画帧数/节奏不足 | 运动质量（挤压拉伸/前摇后摇）优先于贴图精度 |

**整合层（打光/影子/色调/相机/比例纪律）是代码工作，全责在 agent，
不在生图模型。** 用户手里的 image2 生图质量不是瓶颈，禁止以"素材不行"为由搪塞。

## 4. 交付协议（对用户）

用户没有编程与游戏开发经验，只会也只需要说感受。因此：

1. 每次体验类交付必须附：**一段录屏或连续帧** + **一张"你应该能感觉到的变化"清单**
   （每条一句人话，如"怪物脚下黄圈没了"）。
2. 用户只需回答三种话："行" / "不行" / "没感觉"。"没感觉" = agent 返工，
   不是用户不会看。
3. 改动单元要小到可感知。禁止一夜堆十个系统然后报告"做了大量优化"。
4. 写给用户看的内容一律说人话；专业表述只写进 agent 文档（本文件、commit、memory）。

## 5. 执行分工（token 纪律）

- **主 agent（总监）绝不下放**：审美拍板、手感参数、所有截图/录屏的亲眼验收、
  根因诊断、每类效果的第一个标杆实现。
- **派发给 Codex（施工队）**：标杆确立后的规模化铺开（同一光影模式应用到全部
  怪物/道具/UI）、资产管线批量执行、机械性重构。派发任务包必须带明确完成判据
  与禁改范围；产物回传主 agent 亲眼验收，**施工方不许自我认证"效果很好"**。
